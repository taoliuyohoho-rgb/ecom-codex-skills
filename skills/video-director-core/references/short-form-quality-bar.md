# Short-Form Quality Bar

Use this for TikTok or adjacent short-form social video tasks across repos.

This file does not decide:

- account niche
- product facts
- claims truth
- local audience ladder
- local follow promise

Those must come from the local repo or user input.

This file only defines reusable quality bars that should hold across repos.

## 1. Angle

- Lock one audience or use case from local truth before scripting.
- Lock one main job for the clip:
  - stop scroll
  - earn watch-through
  - earn follow intent
  - earn save/share intent
- Prefer extending a repeatable series, hook family, or proof structure before opening a disconnected new angle.

Do not aim at a vague `everyone who might buy / like this`.

## 2. Script

- One clip should carry one clear promise.
- The clip should teach, prove, resolve, or correct one concrete thing.
- For TikTok script drafting or script review, also read `references/tiktok-script-delivery.md`.
- Default informative / corrective / proof short-form scripts to `total -> breakdown -> total` unless the local repo has a better explicit shape.
- The first `3` seconds should already create:
  - motion
  - tension
  - result
  - object read
  - or visible correction
- Score the hook `1-5` for stop-scroll strength before delivery; rewrite if it is below `4/5`.
- By line `2` or roughly second `6`, the viewer should already be entering proof, consequence, or a concrete use scenario.
- Script should make these explicit:
  - hook
  - body
  - proof
  - CTA
  - audio plan

If proof is weak, the answer is usually to tighten the angle or script, not to add filler.

## 3. Format And Execution

- Default to `9:16`.
- Use native `16:9` only when a longer demonstration or instructional structure truly benefits from landscape.
- Do not treat black bars, framed fake-landscape, or edge text as acceptable landscape execution.
- Frame `1-3` should be readable:
  - product
  - person
  - key motion
  - result state
- Lighting and stability should support proof readability.
- Trim dead time aggressively.
- When the format allows it, try to earn `3-5` proof beats instead of relying on one static explanation block.

## 4. Review

- Quality beats raw publishing volume.
- If reach is acceptable but watch-through or follow intent is weak, fix hook clarity, proof density, or early pacing before publishing more of the same.
- Review with the best available evidence:
  - watch time / retention
  - drop-off point
  - saves
  - shares
  - follows / click intent
  - comments that reveal confusion or next questions

Do not judge content quality by views alone.

## 5. Hard Fails

- Audience/use case is vague
- Clip has no concrete value beyond looking nice
- Proof does not match the promise
- First `3` seconds are inert
- Format choice is accidental
- Clip drifts outside local account or product truth
