# TikTok Script Delivery

Use this when the task is a TikTok or adjacent short-form script draft, script review, or publish-ready posting package.

Treat this as a shared delivery contract across repos unless a local repo defines a stricter bar.

## 1. Default Shape

For correction-led, proof-led, buyer-guide, explainer, and most UGC/ecommerce short scripts, default to:

`total -> breakdown -> total`

- `total`:
  open with one sentence that states the claim, correction, or conclusion clearly
- `breakdown`:
  give `2-4` short support beats, usually pain, tradeoff, consequence, or proof
- `total`:
  restate the practical takeaway before the CTA

If a script intentionally breaks this shape, state why.

## 2. Hook Bar

Before showing a script draft, score the hook `1-5` for stop-scroll strength.

- if the score is below `4/5`, rewrite before delivery
- do not hand the user a weak first draft just to ask whether it is strong enough

Common strong hook families:

- correction
- mistake callout
- consequence / loss
- direct audience callout
- comparison with a clear winner

Avoid flat topic intros that only name the subject without tension.

## 3. Early Proof Bar

By line `2` or roughly second `6`, the viewer should already be entering:

- a proof beat
- a concrete consequence
- or a recognizable household / product-use scenario

Do not spend the whole middle on explanation while delaying proof.

## 4. Required Self-Critique

When delivering a script draft, explicitly state:

- whether it is `total -> breakdown -> total`
- hook score `/5`
- why the hook should stop scroll
- the weakest line

If the weakest line is fixable, fix it before asking for approval.

## 5. Default Delivery Pack

Unless a local repo says otherwise, a publish-ready TikTok script package should include:

- `Task`
- `Structure Check`
- `Hook Check`
- `Weakest Line`
- `Script`
- `Posting Pack`

`Posting Pack` defaults to:

- `Title`
- `Cover text`
- `Hashtags`
- optional `Pinned comment`

